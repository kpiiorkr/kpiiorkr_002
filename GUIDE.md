
# 한국프로세스혁신협회 홈페이지 관리 가이드

본 웹사이트는 서버리스(Serverless) 프론트엔드 환경에서 동작하며, 동적 데이터(게시물, 로고 등)를 영구적으로 저장하기 위해 브라우저의 `localStorage`를 사용합니다. Vercel 배포 후 데이터가 사라지거나 화면이 보이지 않을 때 아래 가이드를 따라주세요.

## 1. Vercel 배포 후 화면이 보이지 않을 때
화면이 하얗게 나오는 현상은 주로 **API Key 설정 미비**로 인한 자바스크립트 에러 때문입니다.
1. Vercel 프로젝트 대시보드로 이동합니다.
2. **Settings > Environment Variables** 메뉴를 클릭합니다.
3. `API_KEY`라는 이름으로 사용자의 Google Gemini API Key를 등록합니다.
4. **Deployments** 탭에서 최신 빌드를 다시 실행(Redeploy)합니다.

## 2. 게시물 및 이미지 유지 방법 (Persistence)
본 앱은 별도의 DB가 없으므로 브라우저를 바꾸거나 쿠키를 삭제하면 데이터가 초기화됩니다. 이를 복구하고 유지하는 방법은 다음과 같습니다.

### A. 외부 게시판 실시간 동기화 (권장)
메인 화면의 공지사항, 사회공헌활동 등은 관리자 페이지의 **[AI 실시간 동기화]** 기능을 통해 언제든 외부 사이트(`kpii.or.kr`)에서 최신 데이터를 가져올 수 있습니다.
- 데이터가 사라졌다면 관리자로 로그인 후 **[동기화 시작]** 버튼을 누르세요. AI가 외부 게시판을 자동으로 긁어와 현재 사이트를 업데이트합니다.

### B. 로고 및 배너 이미지 설정
로고나 메인 배너 이미지를 직접 업로드한 경우, 해당 데이터는 브라우저에 저장됩니다. 이를 영구적으로 유지하려면:
- 관리자 페이지에서 이미지를 업로드할 때 **이미지 호스팅 사이트(예: ImgBB, GitHub 등)**에 먼저 올린 후, 해당 이미지 URL을 소스 코드의 `constants.tsx` 또는 관리자 설정에 입력하는 것이 가장 안전합니다.
- 현재는 파일 업로드 시 Base64 형태로 로컬에 저장되므로, 다른 PC에서 접속 시 이미지가 기본값으로 보일 수 있습니다.

## 3. 관리자 정보
- **기본 비밀번호**: `password` (관리자 페이지 하단에서 변경 가능)
- **운영 팁**: 사이트 데이터가 꼬였을 경우 브라우저 개발자 도구(F12)의 Application -> Local Storage에서 `kpii_bbs`와 `kpii_settings`를 삭제하고 새로고침하면 초기 상태로 복구됩니다.
